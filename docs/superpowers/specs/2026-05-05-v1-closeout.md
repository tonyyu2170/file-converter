## 1. /about page

### 1.1 Route

- New `src/app/about/page.tsx`.
- Reachable from the layout footer (the existing site-version line gains a sibling "about" link), and from the sidebar bottom region.
- No drag-drop, no tool framing — static content page rendered with the same brutalist treatment as the rest of the site.

### 1.2 Content

Six sections, in order. Single-column layout, ~720 px max width, monospace body, `[ section ]`-style headers consistent with home and tool pages.

#### The promise
One direct line, set at hero scale: "Files never leave your device."

#### Why this exists
1–2 sentences pulled from master spec §1: common file conversions today require uploading personal documents to ad-supported third-party sites; this site does the conversion entirely in the browser so no file ever traverses the network.

#### Verify it yourself
Numbered steps the user can perform right now in their current browser:

1. Open DevTools (Cmd+Option+I on macOS, F12 on Windows/Linux).
2. Switch to the Network tab.
3. Set the filter to "Fetch/XHR".
4. Drop a file in any tool on this site.
5. Click Convert.
6. Observe: zero new requests during conversion.

Followed by one clarifying paragraph: page-load assets show on first visit and are then cached; nothing fetched during the actual conversion step.

#### How it works
Three bullets:
- **Static export.** No server runtime — the entire site is HTML/JS/CSS/WASM served from a CDN.
- **Web Worker per conversion.** Conversion code runs off the main thread; large files don't freeze the tab.
- **Strict CSP.** `connect-src 'self'` makes off-origin fetches structurally impossible — not a promise, an enforced header.

#### Engines

| `image-bg-remove` | @huggingface/transformers (MODNet model) — *portrait-optimized* | Apache 2.0 |

Implementation detail: the `library` and `license` columns come from new optional fields on the engine descriptor (`EngineMeta`). Pre-existing descriptors gain these fields as part of this phase. The `image-bg-remove` row carries a footnote linking to its design spec note about portrait-only quality.

## 2. Verification sweep

### 2.1 Lighthouse ≥ 95

- Manual run after each deploy via `pnpm build && pnpm preview` in one terminal, then `npx @lhci/cli@latest autorun --collect.url=http://localhost:3000` in another (or the deployed URL post-deploy).
- Target: ≥ 95 in performance, accessibility, best-practices, SEO. Not gated in CI (Lighthouse runs are flaky on CI hardware) but documented as a release checklist.
- Documented in `docs/superpowers/qa-checklist.md` (created in this phase).
- Fix any score < 95 before declaring v1 done. Common offenders to check first: render-blocking JS on the home route, accessible-name violations on icon-only buttons, missing meta description, missing `lang` attribute on `<html>`.

### 2.2 axe AA clean

- New E2E spec `tests/e2e/a11y.spec.ts` using `@axe-core/playwright`.
- Asserts zero AA violations on each of:
  - `/`
  - `/about`
  - `/tools/pdf-merge` (multi-input + drag — exercises StagingArea a11y)
  - `/tools/image-convert` (single-input + options panel — exercises OptionsPanel a11y)
  - `/tools/pdf-edit` (page tray + grid drag — new, exercises rectSortingStrategy + keyboard sensors)
- Runs in default CI alongside the existing Playwright suite. No environment gating — should be cheap and fast.
- Failure surface: the `axe` violation includes `id`, `impact`, `description`, and selector — the spec asserts on the count and snapshots the descriptions for diff-friendliness.

### 2.3 securityheaders.com grade A

- Manual checklist post-deploy. Document the scan URL and required minimum grade in `docs/superpowers/qa-checklist.md`.
- Required headers (set in `vercel.json`, since `next.config.ts` headers don't apply for static export):
  - `Strict-Transport-Security: max-age=31536000; includeSubDomains; preload`
  - `Content-Security-Policy` with `default-src 'self'`, `connect-src 'self'`, `script-src 'self' 'wasm-unsafe-eval'`, `style-src 'self'` (no `'unsafe-inline'`), `img-src 'self' data: blob:`, `worker-src 'self' blob:`, `frame-ancestors 'none'`
  - `X-Frame-Options: DENY`
  - `X-Content-Type-Options: nosniff`
  - `Referrer-Policy: strict-origin-when-cross-origin`
  - `Permissions-Policy: camera=(), microphone=(), geolocation=(), interest-cohort=()`
- If the current `vercel.json` is missing any of these, this phase adds them. If `'unsafe-inline'` has crept into `style-src` (via shadcn or runtime tokens — flagged in master spec §18 as an open risk), the build is fixed, not the header.

### 2.4 Bundle isolation gate

- New `scripts/check-bundle-isolation.mjs`:
  1. Runs after `pnpm build` (wired as `pnpm postbuild` in `package.json`).
  2. Walks the Next.js build output for the homepage route (`out/` for static export, or `.next/server/app/page.js` from the build manifest pre-export — whichever is canonical for the project's static-export setup).
  3. Asserts that no module path under `src/engines/<engineId>/` (i.e., not `_shared/`) is referenced from the homepage chunk.
  4. Allows references to `src/engines/_shared/*` (ToolFrame, types, harness — these *are* part of the homepage shell).
  5. Exits non-zero with an actionable error message listing the offending engine paths if any are found.
- Wired as a CI gate (failing on violation, per the design discussion). The script is small enough to run on every CI build with no measurable cost.

### 2.5 Deploy validation

- Post-deploy checklist documented in `docs/superpowers/qa-checklist.md`. Each item is a `curl -I` or browser DevTools check against the production URL:
  - `Cache-Control` on `.wasm` assets: `public, max-age=31536000, immutable` (long-lived; build hashes handle invalidation).
  - CSP header is the one in §2.3 — no `'unsafe-inline'` or `'unsafe-eval'` slipped in via Vercel defaults.
  - `Strict-Transport-Security` `max-age` ≥ 31536000.
  - HTTP → HTTPS redirect is in place.
  - Conversion of a representative file (HEIC, PDF merge) produces zero outbound network requests in the Network panel — the manual analog of the §10.3 privacy claim.

### 3.4 §16 (Future scope — post-v1)

Add bullets:

- "PDF → DOCX. Cut from v1 because best-effort layout reconstruction does not meet the project quality bar. Revisit when a permissively-licensed in-browser solution exists with materially better fidelity than mammoth-style structural mapping."
- "image-bg-remove model swap. The Phase 16 model is portrait-optimized MODNet (per its design spec's '2026-05-04 update' section), which produces unusable masks on non-portrait inputs.
